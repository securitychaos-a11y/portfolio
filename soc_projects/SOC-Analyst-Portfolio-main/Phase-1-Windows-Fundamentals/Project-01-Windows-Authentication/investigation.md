# Investigation

## Objective

Determine whether multiple failed logon events represent malicious activity or legitimate user behavior.

---

## Evidence Collected

Windows Security Log

- Event ID 4625
- Event ID 4624
- Event ID 4672

Sysmon

- Installed for future investigations.

Splunk

- Authentication events successfully ingested.

---

## Findings

### Event ID 4625

Observed:

- Interactive Logon
- Logon Type 2
- Source Address 127.0.0.1
- Caller Process

```
C:\Windows\System32\svchost.exe
```

Failure Reason

```
An Error occurred during Logon
```

---

### Event ID 4624

Successful authentication immediately after three failed attempts.

---

### Event ID 4672

Administrative privileges assigned to authenticated account.

Observed privileges include

- SeDebugPrivilege
- SeBackupPrivilege
- SeRestorePrivilege
- SeTakeOwnershipPrivilege
- SeImpersonatePrivilege

---

## Analysis

Evidence indicates:

- Authentication originated locally.
- Authentication performed interactively.
- Authentication generated by legitimate Windows components.
- Timeline matches user intentionally entering incorrect password before successful login.

No indicators of malicious authentication activity were identified.

---

## Final Conclusion

Classification:

False Positive

Reason:

Expected user behavior generated expected Windows authentication events.
